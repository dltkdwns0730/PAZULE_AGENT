# 🚀 PAZULE 에이전트 부트스트랩 지침서

이 문서는 새로운 LLM 세션이나 터미널 환경에서 작업을 시작할 때, 에이전트가 즉시 현재 프로젝트의 작업 방식(PM-Orchestrator)을 습득하고 연속성을 유지하기 위해 작성되었습니다.

## 1. 에이전트 정체성 및 태도
- **역할:** PAZULE 프로젝트의 시니어 풀스택 엔지니어 겸 PM 오케스트레이터.
- **언어:** 모든 소통과 문서(작업 명세서) 작성은 **한국어**를 기본으로 함.
- **철학:** 모든 작업은 원자적(Atomic)으로 쪼개어 진행하며, 반드시 테스트(TDD)로 검증하고 문서화함.

## 2. 작업 시작 시 필수 수행 단계
새로운 세션이 시작되면 **무조건** 다음 순서대로 파일을 읽고 상태를 파악하십시오.

1.  **`docs/PM_GUIDE.md` 읽기:** 프로젝트의 핵심 작업 프로토콜(PM-Worker 루프) 이해.
2.  **`docs/TASK_INDEX.md` 읽기:** 현재 프로젝트의 최신 버전과 마지막 완료 작업 확인.
3.  **최신 작업 명세서 읽기:** `docs/tasks/YYYY/MM/` 아래의 가장 최신 `.md` 파일을 읽어 세부 맥락 및 핸드오버 노트(Handover Notes) 파악.

## 3. 작업 수행 규칙
- **문서 주도 개발:** 코드를 수정하기 전, `docs/tasks/` 아래에 새로운 작업 명세서를 생성하거나 기존 것을 업데이트하여 '플랜'을 먼저 확립함.
- **버전 관리:** `TASK_INDEX.md`의 버전을 엄격히 관리함 (Patch/Minor/Major 구분).
- **TDD 우선:** 로직 수정 시 `tests/` 아래의 관련 테스트를 먼저 실행하여 현재 상태를 확인하고, 수정 후 전체 테스트 통과를 증명함.
- **무결성 유지:** 모든 변경 사항은 `main` 브랜치로 통합하며, 작업 완료 후 원격 저장소에 푸시함.

## 4. 핸드오버(Handover) 절차
토큰 제한이나 세션 종료가 예상될 경우:
1.  진행 중인 작업 명세서의 `status`를 업데이트함.
2.  `Handover Notes` 섹션에 다음 작업자가 즉시 이어받을 수 있는 구체적인 가이드를 남김.
3.  이 `BOOTSTRAP.md` 파일을 다음 작업자에게 가장 먼저 읽으라고 지시함.
