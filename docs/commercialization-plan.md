# PAZULE Commercialization Plan

이 문서는 현재 제품 구조 기준의 B2B2C 상용화 검토 문서입니다.

## Product Position

PAZULE은 오프라인 방문 인증, AI 사진 미션 판정, 쿠폰 발급/사용, 기업별 관리자 리포팅을 하나의 흐름으로 연결합니다.

```text
방문자
  -> OAuth 로그인
  -> 현장 GPS 기반 미션 시작
  -> 사진 제출
  -> AI 판정
  -> 쿠폰 수령/사용

기업/파트너
  -> 기업별 관리자 콘솔
  -> 미션 로그/쿠폰/사용자 활동 조회
  -> 쿠폰 리딤 처리
```

## Current Product Surface

| 영역 | 현재 상태 |
|---|---|
| 사용자 앱 | 로그인, 오늘의 힌트, 미션 제출, 결과, 쿠폰 지갑 |
| 관리자 콘솔 | 대시보드, 미션 로그, 미션 상세, 쿠폰 관리, 사용자 조회 |
| 인증 | Supabase OAuth/JWT |
| 데이터 분리 | 기업, 사이트, 조직 멤버, 미션 세션, 쿠폰 관계 |
| AI 판정 | SigLIP2 기본, BLIP 앙상블, Qwen VL fallback |
| 저장소 | 기본 JSON, DB backend 선택 가능 |

## Commercial Value

| 대상 | 가치 |
|---|---|
| 로컬 매장/관광지 | 실제 방문 기반 쿠폰 발급으로 무분별한 프로모션 비용을 줄임 |
| 브랜드/운영사 | 기업별 미션 참여, 쿠폰 발급, 사용 현황을 운영 콘솔에서 확인 |
| 방문자 | 현장에서 미션을 수행하고 즉시 보상 경험을 받음 |
| 플랫폼 운영자 | 마스터 권한으로 전체 기업과 사용자 활동을 모니터링 |

| 캠페인 패키지 | 장소/이벤트별 미션 구성과 운영 대행 |
| 데이터 리포트 | 방문 인증, 미션 완료, 쿠폰 사용 데이터를 집계 리포트로 제공 |

## Current Technical Readiness

| 항목 | 현재 근거 |
|---|---|
| 인증 소유권 | 서버가 Supabase JWT `sub`를 기준으로 미션 소유자 검증 |
| 기업 스코프 | `organizations`, `sites`, `organization_members` 기반 관리자 범위 |
| 미션 안정성 | TTL `60`분, 최대 제출 `3`회, GPS 반경 `300m`, GPS 정확도 `100m` |
| AI 임계값 | 위치 `0.70`, 분위기 `0.35` |
| 쿠폰 운영 | 미션 ID 기반 쿠폰 발급과 관리자 리딤 API |
| 검증 | pytest, Vitest, ruff, ESLint, GitHub Actions |

## Operating Metrics

| 지표 | 의미 |
|---|---|
| 미션 시작 수 | 방문자가 현장에서 미션을 시작한 수 |
| 미션 완료율 | 제출 대비 AI 판정 성공 비율 |
| 쿠폰 발급 수 | 성공 미션에서 발급된 쿠폰 수 |
| 쿠폰 사용률 | 발급 쿠폰 대비 리딤 처리 비율 |
| 관리자 조회 범위 오류 | 기업 스코프 밖 데이터 노출 여부 |
| AI fallback 비율 | Qwen VL 등 비용성 fallback 호출 비율 |

정량 목표는 실제 파트너 파일럿 데이터가 쌓인 뒤 별도 운영 지표 문서에 기록합니다.
