# v3.4.1 Login Entry & Profile UI Consistency

## Purpose

로컬 실행 시 비인증 사용자가 `http://localhost:5173/`로 진입해도 로그인 화면으로 안정적으로 이동하게 하고, 로그인/프로필 화면의 주요 액션 UI가 서로 다른 형태로 보이던 문제를 정리한다.

## Scope

- `/login` 실제 Google 로그인, 사용자 데모 로그인, 관리자 데모 로그인 UI 정렬
- `/profile` 쿠폰 지갑 보기와 위험 영역 액션 카드 정렬
- 비인증 루트 진입 라우팅 회귀 테스트
- 로그인 복구 과정에서 확인된 API 루트 404 로그와 Supabase JWT 오류 응답 안정화

## Changes

- `Login` 화면의 3개 로그인 진입점을 동일한 카드/버튼 구조로 통일했다.
- `Profile` 화면에 공통 액션 카드 패턴을 추가해 `쿠폰 지갑 보기`와 `위험 영역`의 여백, 버튼 높이, shadow, border radius를 맞췄다.
- `/login`, `/admin/login`, `/auth/callback`, `/onboarding`에서 하단 내비게이션을 숨겨 로그인/콜백 화면이 하단 바와 충돌하지 않게 했다.
- 백엔드 `GET /`는 API 안내 JSON을 반환하고 `GET /favicon.ico`는 204로 응답해 로컬 브라우저 직접 접속 시 불필요한 404 로그를 줄였다.
- Supabase JWKS 조회나 JWT decode 실패는 `invalid_or_expired_token` 401 응답으로 감싸 클라이언트 인증 오류가 서버 예외로 새지 않게 했다.
