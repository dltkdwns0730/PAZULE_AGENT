# 📋 작업 명세서: 에이전트 부트스트랩 지침서 작성

## 1. 배경 및 목적
- **배경:** 새로운 세션이나 다른 LLM 서비스에서 작업을 시작할 때마다 현재의 작업 방식(PM-Worker)을 수동으로 설명해야 하는 번거로움이 있음.
- **목적:** 모든 작업 규칙과 컨텍스트 복구 절차를 담은 `BOOTSTRAP.md`를 생성하여, 새로운 에이전트가 즉시 현재 프로젝트의 핵심 일원으로 합류할 수 있도록 함.

## 2. 구현 세부 사항
- [x] `docs/BOOTSTRAP.md` 생성:
    - 에이전트의 페르소나 및 한국어 소통 규칙 명시.
    - `PM_GUIDE` -> `TASK_INDEX` -> `최신 명세서` 순의 필수 읽기 루프 정의.
    - 작업 수행 및 핸드오버의 표준 절차 정립.

## 3. 검증 결과 및 품질
- **확인:** `BOOTSTRAP.md` 파일이 프로젝트 루트의 `docs/` 폴더에 정상 생성됨.
- **확인:** 지침서의 내용이 명확하고 실질적인 액션 아이템 위주로 구성되어 타 LLM이 이해하기 용이함.

## 4. 핸드오버 참고 사항
- **사용 방법:** 새로운 창에서 "docs/BOOTSTRAP.md 읽고 시작해"라고 말하기만 하면 모든 세팅이 완료됨.
