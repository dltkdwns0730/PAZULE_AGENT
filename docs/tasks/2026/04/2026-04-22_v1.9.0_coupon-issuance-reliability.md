# 📋 작업 명세서: 쿠폰 발급 로직 강화 및 지갑 연동 최종 조치

## 1. 배경 및 목적
- **배경:** 지속적인 조치에도 불구하고 지갑 페이지에서 쿠폰 노출이 간헐적으로 누락되는 현상 발생. 조사 결과 프론트엔드 API 호출 시 사용자 ID 전달 방식에 일관성이 부족함을 발견.
- **목적:** 프론트엔드-백엔드 간 사용자 ID 전달 체계를 전면 재정비하여 지갑 데이터 동기화의 100% 신뢰성 확보.

## 2. 구현 세부 사항
- [x] **프론트엔드 API 서비스 (`api.js`):** `issueCoupon` 메서드가 `user_id`를 인자로 받아 백엔드로 명시적 전송하도록 인터페이스 확장 완료.
- [x] **커스텀 훅 (`useMission.js`):** 쿠폰 발급 요청 시 `guest` 사용자 ID를 기본값으로 명시하여 호출하도록 로직 보강 완료.
- [x] **백엔드 라우트 (`routes.py`):** 
    - `/api/coupon/issue` 엔드포인트가 요청 본문의 `user_id`를 최우선으로 사용하도록 수정 완료.
    - 세션에 저장된 ID와 요청 ID를 교차 검증하여 데이터 누락 가능성 차단 완료.

## 3. 검증 결과 및 품질
- **확인:** 신규 미션 완료 후 쿠폰 발급 시, `coupons.json`에 정확한 `user_id`가 기록됨을 확인.
- **확인:** 지갑 페이지 진입 시 해당 `user_id`를 기반으로 한 필터링이 서버와 클라이언트 양측에서 정합성 있게 동작함을 확인.
- **확인:** 이제 미션 성공 -> 발급 -> 지갑 확인으로 이어지는 사용자 여정에서 데이터가 유실되지 않음.

## 4. 핸드오버 참고 사항
- **주의:** 본 조치는 신규 발급되는 쿠폰부터 적용되며, 테스트 과정에서 생성된 구버전 데이터는 초기화 후 테스트를 권장함.
