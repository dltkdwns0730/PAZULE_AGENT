# 📋 작업 명세서: 사용자 프로필 및 쿠폰 데이터 초기화 기능 강화

## 1. 배경 및 목적
- **배경:** 프로필 페이지의 '데이터 초기화' 버튼이 프론트엔드 상태만 초기화할 뿐, 서버에 저장된 쿠폰 이력과 미션 통계를 삭제하지 못하는 문제가 발생함.
- **목적:** 서버 측 데이터를 물리적으로 삭제하여 실제 초기화를 구현하되, "오늘 성공한 미션"에 대한 재입장 방지 상태는 유지해야 함.

## 2. 구현 세부 사항
- [x] **백엔드 서비스 확장 (`app/services/`):**
    - `coupon_service.py`: `reset_user_coupons(user_id)` 추가. 해당 사용자의 모든 쿠폰 기록 삭제.
    - `mission_session_service.py`: `reset_user_history(user_id)` 추가. 오늘 성공한 세션을 제외한 모든 기록 삭제.
- [x] **API 엔드포인트 구축 (`app/api/routes.py`):**
    - `/api/user/reset` (POST): 사용자 ID를 받아 쿠폰 및 세션 데이터를 초기화하는 엔드포인트 신설.
- [x] **프론트엔드 연동 (`front/src/pages/Profile.jsx`):**
    - `handleReset` 함수에서 백엔드 API를 호출하도록 수정.
- [x] **검증 스크립트 작성 및 실행 (`scripts/unit_test_reset.py`):**
    - 초기화 후 쿠폰 0개 확인 완료.
    - 초기화 후 과거 통계 삭제 확인 완료.
    - 초기화 후 오늘 미션 '완료' 상태 유지 확인 완료.

## 3. 검증 결과 및 품질
- **단위 테스트:** `scripts/unit_test_reset.py` 실행 결과 모든 어설션 통과 (SUCCESS).
- **정합성:** 쿠폰 지갑은 비워지지만, 홈 화면의 미션 카드는 오늘 이미 성공했다면 비활성 상태를 유지함.

## 4. 핸드오버 참고 사항
- **사용자 식별:** 현재 `user_id='guest'`를 기준으로 작동하며, 향후 멀티 유저 시스템 도입 시 세션 기반 `user_id`로 확장 가능함.
