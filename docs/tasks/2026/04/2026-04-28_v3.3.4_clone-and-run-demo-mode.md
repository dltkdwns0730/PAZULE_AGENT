# v3.3.4 Clone-and-Run Demo Mode

## Purpose

면접관과 리뷰어가 Supabase, OAuth, AI provider 비밀값 없이도 로컬에서 사용자 앱과 관리자 콘솔 흐름을 확인할 수 있도록 데모 실행 경로를 제공한다.

## Scope

- Backend demo bearer token support
- Frontend demo user/admin login buttons
- `.env.example`
- README and config documentation
- Unit tests for demo auth/session helpers

## Current Demo Behavior

| 항목 | 값 |
|---|---|
| `DEMO_AUTH_ENABLED` | `development=True`, `test/production=False` |
| User token | `demo-user-token` |
| Admin token | `demo-admin-token` |
| Admin role | `platform_master` |
| Storage | `STORAGE_BACKEND=json` |
| Model validation | `BYPASS_MODEL_VALIDATION=true` in development |
| Metadata validation | `SKIP_METADATA_VALIDATION=true` in development |

## Validation

- `tests/security/test_auth_demo.py`
- `front/src/test/services/supabaseAuth.test.js`
