# 📋 작업 명세서: 지갑 페이지 데이터 연동 및 쿠폰 조회 기능 구축

## 1. 배경 및 목적
- **배경:** 지갑 페이지(`CouponWallet.jsx`)가 하드코딩된 임의의 데이터를 보여주고 있어 실제 발급된 쿠폰과 동기화되지 않음.
- **목적:** 실제 쿠폰 발급 파이프라인과 지갑 UI를 연동하여 사용자가 획득한 진짜 혜택만 확인할 수 있도록 개선.

## 2. 구현 세부 사항
- [x] `app/services/coupon_service.py`: 사용자 ID를 기반으로 발급된 모든 쿠폰을 조회하는 `get_user_coupons` 메서드 추가 완료.
- [x] `app/api/routes.py`: `/api/coupons` REST 엔드포인트 신규 구축 완료.
- [x] `front/src/pages/CouponWallet.jsx`: 
    - 하드코딩된 가짜 쿠폰 데이터 리스트 제거 완료.
    - 컴포넌트 마운트 시 `/api/coupons` API를 호출하여 동적 데이터 렌더링 구현 완료.
    - 쿠폰 상태(사용 완료, 만료 등)에 따른 시각적 스타일 처리 보완 완료.
    - 쿠폰이 없을 경우의 "Empty State" UI 추가 완료.

## 3. 검증 결과 및 품질
- **확인:** 지갑 페이지 진입 시 로딩 스피너 노출 후 실제 발급된 쿠폰 리스트가 정상적으로 표시됨.
- **확인:** 쿠폰 클릭 시 해당 쿠폰 정보가 `useMissionStore`에 저장되고 스캔 페이지로 정상 이동함.
- **확인:** 백엔드에서 `mission_sessions.json`과 `coupons.json`을 교차 검증하여 특정 사용자의 쿠폰만 정확히 필터링함.

## 4. 핸드오버 참고 사항
- **주의 사항:** 현재 데모용으로 `user_id=guest` 고정값을 사용하고 있으나, 향후 로그인 시스템 도입 시 동적 사용자 ID로 전환 가능하도록 API가 설계됨.
