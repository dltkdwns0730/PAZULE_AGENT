# 📋 작업 명세서: 분위기 미션 성능 최적화 및 힌트 시스템 연동

## 1. 배경 및 목적
- **배경:** '분위기 담아내기' 미션의 판정 정확도가 낮아 사용자 경험이 저하됨. 또한, 실패 시 제공되어야 할 AI 생성 힌트가 프론트엔드 UI에 정상적으로 노출되지 않음.
- **목적:** 앙상블 모델(SigLIP2, BLIP)의 판정 로직을 정밀화하고, 실제 데이터를 통한 벤치마크 결과를 바탕으로 임계값을 조정하며, 실패 시 UI에 AI 힌트를 표시함.

## 2. 구현 세부 사항

### 2.1 모델 판정 로직 개선
- [x] `app/models/siglip2.py`: SigLIP2의 점수 계산 방식을 단순 sigmoid에서 `match` vs `mismatch` 후보군 간의 **softmax** 방식으로 변경. (상대적 확신도 강화, 벤치마크 점수 0.0001 -> 0.3~0.5 수준으로 상승)
- [x] **벤치마크 수행 (`scripts/benchmark_atmosphere.py`):** 실제 에셋(나남출판사, 지혜의 숲 등)을 대상으로 SigLIP2와 BLIP의 응답 수치를 정량화함.

### 2.2 힌트 시스템 UI 연동
- [x] `front/src/pages/PhotoSubmission.jsx`: 미션 실패 시 서버에서 반환한 `hint` (LLM 생성 힌트)를 상단 '인증 실패' 배너 내에 팁 형태로 출력하도록 UI 개선.
- [x] **백엔드 검증 (`scripts/debug_hint_flow.py`):** 의도적 실패 시나리오를 통해 LLM 힌트 생성 및 API 응답 포함 여부 확인 완료.

## 3. 검증 결과 및 품질
- **SigLIP2 스코어링:** Softmax 도입 후 정답군에 대해 유의미한 변별력(0.3~0.5)을 갖게 됨을 벤치마크로 증명.
- **힌트 흐름:** 실패 시 Gemini/GPT 기반의 은유적 힌트가 프론트엔드 배너에 정상 노출됨을 확인.

## 4. 핸드오버 참고 사항
- 현재 `ATMOSPHERE_PASS_THRESHOLD`가 0.35~0.4 수준으로 조정될 필요가 있음 (현재 0.2로 설정되어 있다면 다소 낮을 수 있음). `app/core/config/settings.py` 확인 권장.
