# 📋 작업 명세서: 사용자 프로필 및 통계 시스템 구축

## 1. 배경 및 목적
- **배경:** 홈 화면의 프로필 버튼이 동작하지 않으며, 사용자의 탐험 성과를 확인할 수 있는 공간이 부재함.
- **목적:** 사용자의 미션 성공 통계와 히스토리를 제공하여 앱 사용의 동기를 부여하고 탐험가 정체성을 강화함.

## 2. 구현 세부 사항
- [x] `front/src/pages/MissionHome.jsx`: 프로필 버튼 네비게이션 오류 수정 완료.
- [x] `app/services/mission_session_service.py`: 사용자 ID별 미션 통계 산출 함수(`get_user_stats`) 추가 완료.
- [x] `app/api/routes.py`: `/api/user/stats` 엔드포인트 신규 구축 완료.
- [x] `front/src/pages/Profile.jsx`: 신규 프로필 페이지 UI 구현 (Stats, History, Rewards 섹션) 완료.
- [x] `front/src/App.jsx`: `/profile` 라우트에 신규 Profile 컴포넌트 연결 완료.

## 3. 검증 결과 및 품질
- **확인:** 홈 화면 프로필 아이콘 클릭 시 프로필 페이지로 정상 이동함.
- **확인:** 백엔드에서 집계된 실제 통계 데이터(총 시도, 성공 횟수)가 UI에 정확히 렌더링됨.
- **확인:** 데이터 초기화 버튼 동작 및 최근 5건의 미션 히스토리 표시 확인.

## 4. 핸드오버 참고 사항
- **통계 항목:** 총 시도 횟수, 랜드마크 성공, 분위기 성공, 획득 쿠폰 수를 포함함.
- **사용자 식별:** 현재는 데모 목적으로 `user_id=guest` 고정값을 사용함.
