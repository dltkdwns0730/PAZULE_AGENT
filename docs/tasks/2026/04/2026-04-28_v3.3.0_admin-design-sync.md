# v3.3.0 Admin Design Sync & AdminLogin 제거

## Background
`/admin/login`과 `/admin` 콘솔 페이지가 메인 화면(`/login`)과 완전히 다른 디자인 언어(dark slate 테마)를 사용하고 있다.
또한 `/admin/login`은 일반 `/login`과 동일한 `startGoogleOAuth()` 플로우를 호출하므로 중복 구현이다.

## Goals
1. `/admin/login` 제거 → `AdminRoute`가 `/login`으로 리다이렉트
2. `AdminDashboard`와 `AdminMissionDetail`의 디자인을 PAZULE 토큰으로 동기화
   - `slate-*` 색상 → `dark-teal`, `gray-*`, `warm-cream`, `light-grey`
   - `dark:` variant 제거 (메인 앱과 동일하게 라이트 모드만)
   - `shadow-sm` → `shadow-soft`
   - 하드코딩된 mock 사용자 → `useAuthStore`에서 실제 email 표시
   - 로그아웃 버튼 실제 동작 연결 (`clearSession` + `/login` 이동)

## Changes
- **DELETE** `front/src/pages/AdminLogin.jsx`
- **MODIFY** `front/src/components/AdminRoute.jsx` — redirect to `/login`
- **MODIFY** `front/src/App.jsx` — remove AdminLogin import/route
- **MODIFY** `front/src/pages/AdminDashboard.jsx` — PAZULE 디자인 토큰 적용
- **MODIFY** `front/src/pages/AdminMissionDetail.jsx` — 동일 처리
