# v3.1.0 Admin Operations Console

## Background
Supabase Google OAuth와 `app_metadata.role = admin` 기반 관리자 로그인은 가능해졌지만, `/admin` 화면은 정적 목업이고 사이드바 링크가 `#`로 되어 있어 실제 운영 목록으로 이동할 수 없었다.

## Goal
관리자 계정이 미션 로그, 쿠폰, 유저 상태를 실제 API 기반으로 조회하고 쿠폰 사용 처리를 수행할 수 있게 한다.

## Proposed Change
- Supabase JWT의 `app_metadata.role`을 백엔드 관리자 권한 기준으로 사용한다.
- `/api/admin/*` 운영 조회 API를 추가한다.
- `/admin`, `/admin/logs`, `/admin/logs/:missionId`, `/admin/coupons`, `/admin/users` 라우트를 React Router 기반으로 구성한다.
- 관리자 화면의 `href="#"` 정적 이동을 제거한다.

## Validation Plan
- 관리자 role 없음: `/api/admin/*` 403.
- 관리자 role 있음: summary/logs/detail/coupons/users 조회 가능.
- `/admin#` 이동 없이 사이드바 메뉴가 실제 라우트로 이동.
- 프론트 빌드와 관련 테스트를 실행한다.

## Rollback
관리자 API route와 관리자 프론트 라우트 추가분을 되돌리면 기존 사용자 미션/OAuth 흐름은 유지된다.
