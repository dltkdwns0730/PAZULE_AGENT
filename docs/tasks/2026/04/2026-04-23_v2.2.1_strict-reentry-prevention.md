# 📋 작업 명세서: 미션 재입장 불가 로직 및 UI 최종 보완

## 1. 배경 및 목적
- **배경:** 미션 성공 및 쿠폰 발급 후에도 홈 화면에서 해당 미션이 활성화된 상태로 남아있어 중복 시도가 가능한 문제 해결 필요.
- **목적:** 프론트엔드와 백엔드 간 사용자 ID 전달 체계를 통일하고, 완료된 미션에 대한 시각적/기능적 차단을 완벽히 구현함.

## 2. 구현 세부 사항
- [x] **프론트엔드 API (`api.js` & `useMission.js`):** `getTodayHint` 호출 시 `user_id`를 파라미터로 명시적 전달하도록 수정 완료.
- [x] **백엔드 라우트 (`routes.py`):** 
    - `/get-today-hint`: 당일 성공 기록 또는 쿠폰 보유 여부를 종합하여 `completed` 상태 반환 완료.
    - `/api/mission/start`: 미션 시작 시점에 다시 한번 오늘 완료 여부를 검증하여 부정한 진입 차단 완료.
- [x] **프론트엔드 UI (`MissionHome.jsx`):** 
    - 완료된 미션 카드에 '오늘의 미션 완료!' 오버레이를 표시하고 시각적으로 비활성화(`opacity-70`) 완료.
    - `pointer-events-none`을 적용하여 클릭 핸들러를 원천 차단 완료.

## 3. 검증 결과 및 품질
- **확인:** 미션 성공 후 홈 화면으로 돌아왔을 때, 해당 미션 카드가 즉시 비활성화 상태로 변하며 클릭이 불가능함을 확인.
- **확인:** 서버 데이터베이스(`mission_sessions.json`)와 연동되어 정확한 완료 상태가 UI에 반영됨을 확인.

## 4. 핸드오버 참고 사항
- **주의:** 현재 모든 사용자 식별은 `guest`를 기준으로 동작하며, 추후 개별 로그인 기능 도입 시 `user_id` 파라미터만 변경하면 즉시 호환 가능함.
