# v3.3.5 Environment Template Cleanup

## Purpose

`.env`와 `.env.example`의 역할을 분리한다. 로컬 `.env`는 데모 실행에 필요한 활성 값만 남기고, `.env.example`은 선택 가능한 전체 설정과 설명을 제공한다.

## Current Local `.env`

| 항목 | 현재값 |
|---|---|
| `PAZULE_ENV` | `development` |
| `DEMO_AUTH_ENABLED` | `true` |
| `STORAGE_BACKEND` | `json` |
| `SKIP_METADATA_VALIDATION` | `true` |
| `BYPASS_MODEL_VALIDATION` | `true` |
| `LOCATION_PASS_THRESHOLD` | `0.70` |
| `ATMOSPHERE_PASS_THRESHOLD` | `0.35` |

## Cleanup Rules

- 실제 코드나 스크립트에서 읽지 않는 변수는 `.env`에서 제거한다.
- 선택형 변수는 `.env.example`에서 주석 처리해 빈 문자열이 기본값을 덮어쓰지 않게 한다.
- 실서비스 비밀값은 `.env.example`에 넣지 않는다.
- 현재 임계값은 위치 `0.70`, 분위기 `0.35`를 기준으로 한다.
