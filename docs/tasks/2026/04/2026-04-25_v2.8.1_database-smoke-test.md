# Task Specification: Database Smoke Test v2.8.1

## 1. Context & Objective
- **Context:** v2.8.0에서 SQLAlchemy/Alembic/Supabase Auth 기반은 추가됐지만, 실제 DB 연결 가능 여부를 애플리케이션 코드에서 검증하는 작은 스모크 테스트 경계가 없다.
- **Objective:** 외부 Supabase 자격 증명 없이도 로컬/CI에서 검증 가능한 DB 연결 스모크 유틸과 테스트를 추가하고, 실제 Supabase 주입 시 같은 경계를 재사용할 수 있게 한다.

## 2. Implementation Details
- [x] `session_scope`를 실제 context manager로 사용할 수 있게 정리한다.
- [x] SQLAlchemy `SELECT 1` 기반 DB 스모크 체크 함수를 추가한다.
- [x] SQLite URL을 사용한 로컬 단위 테스트로 연결 체크와 세션 commit/rollback 경계를 검증한다.
- [x] 다음 Supabase 실연결 단계의 남은 작업을 문서화한다.

## 3. Verification & Quality
- **Tests:** `$env:PYTHONPATH='.'; pytest tests/core/test_platform_settings.py tests/db/test_session.py`, `uv run ruff check app/db/session.py tests/db/test_session.py`
- **Validation:** SQLite 기반 테스트 5개가 통과하여 DB URL 주입, SQLAlchemy 연결 체크, session commit/rollback 경계가 동작함을 확인했다. 실제 Supabase 네트워크 연결은 환경변수 주입 후 별도 실행한다.

## 4. Handover Notes
- **Next Steps:** 실제 Supabase `DATABASE_URL`을 주입한 환경에서 `smoke_check_database_connection()`을 호출하는 운영 전용 검증 명령을 추가한다. 이후 `MissionSessionService`와 `CouponService` repository adapter 전환을 별도 작업으로 진행한다.
- **Critical Info:** 현재 런타임 서비스는 여전히 JSON 저장소를 사용한다. 이번 작업은 DB 연결 경계 검증만 다루며 repository adapter 전환은 후속 작업으로 분리했다.
