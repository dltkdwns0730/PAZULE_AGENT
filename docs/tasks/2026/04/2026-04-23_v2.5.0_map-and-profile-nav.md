# 📋 작업 명세서: 메인 화면 내비게이션 복구 및 복합 지도(Map) 페이지 구현

## 1. 배경 및 목적
- **배경:** `Layout.jsx` 하단 내비게이션 바의 'Profile' 버튼과 'Map' 버튼이 빈 클릭 핸들러로 방치되어 작동하지 않는 버그가 보고됨.
- **목적:** Profile 내비게이션을 활성화하고, 사용자가 요구한 두 가지 형태의 지도(야외 지역 웹 지도 + 실내 도면 이미지)를 전환하며 볼 수 있는 새로운 `/map` 페이지를 구현하여 탐험 경험을 강화함.

## 2. 구현 범위 및 세부 태스크

### 2.1 하단 내비게이션 복구
- [x] `front/src/components/Layout.jsx`: 하단 내비게이션의 `Profile` 아이콘 클릭 시 `/profile`로 라우팅되도록 `navigate('/profile')` 추가.
- [x] `front/src/components/Layout.jsx`: 하단 내비게이션의 `Map` 아이콘 클릭 시 `/map`으로 라우팅되도록 `navigate('/map')` 추가.
- [x] 현재 활성화된 탭에 대한 스타일 처리를 `location.pathname`을 기반으로 동적 적용.

### 2.2 신규 지도(Map) 페이지 구축
- [x] `front/src/pages/Map.jsx` 컴포넌트 신규 생성.
- [x] 상단 탭 메뉴(Tab UI) 구현: `야외 지도` vs `실내 도면` 전환 기능.
- [x] **야외 지도 (Web Map API):**
  - 파주 출판단지를 기본 중심으로 하는 구글 지도 임베드(iframe)를 통해 실제 위치 정보 제공.
- [x] **실내 도면 (Image Viewer):**
  - `react-zoom-pan-pinch` 라이브러리를 설치하여, 실내 도면(이미지)을 띄우고 모바일 제스처로 확대/축소 및 패닝(Pan & Zoom)이 가능하도록 구현.
- [x] `front/src/App.jsx`에 `<Route path="/map" element={<Map />} />` 라우트 추가.

## 3. 검증 결과
- **내비게이션:** 하단 바의 Home, Map, Wallet, Profile 모든 버튼이 정상적으로 해당 페이지로 이동함.
- **지도 기능:** `/map` 페이지에서 야외 지도(구글 맵)와 실내 도면(확대 가능한 이미지)이 탭 전환을 통해 정상적으로 표시됨.
- **사용자 경험:** 실내 도면에서 핀치 줌 및 드래그가 매끄럽게 동작함을 확인.
