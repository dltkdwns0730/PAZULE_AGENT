# Task Specification: Platform Foundation v2.8.0

## 1. Context & Objective
- **Context:** PAZULE는 현재 JSON 파일 기반 저장소와 `guest` 사용자 식별을 사용한다. 미션, 쿠폰, 지갑, GPS 검증, 관리자 UI 초안은 구현되어 있으나 B2B 운영에 필요한 DB, Auth, 권한, 감사 로그 기반이 없다.
- **Objective:** Supabase Postgres/Auth로 전환할 수 있도록 설정, ORM 모델, migration 골격, JWT 검증 유틸을 추가한다. 기존 JSON 런타임 동작은 유지한다.

## 2. Implementation Details
- [x] 플랫폼 의존성 선언: SQLAlchemy, Alembic, psycopg, PyJWT.
- [x] DB/Auth 설정 추가: `DATABASE_URL`, `SUPABASE_URL`, `SUPABASE_JWKS_URL`, `SUPABASE_JWT_AUDIENCE`, `SUPABASE_SERVICE_ROLE_KEY`.
- [x] SQLAlchemy 세션/엔진 헬퍼와 B2B 운영 데이터 모델 추가.
- [x] Alembic migration 환경과 초기 platform foundation migration 추가.
- [x] Supabase JWT bearer token 추출 및 검증 유틸 추가.
- [x] 다음 작업에서 기존 JSON 서비스의 repository adapter를 DB 구현으로 교체할 수 있도록 경계 지점을 문서화한다.

## 3. Verification & Quality
- **Tests:** `pytest tests/core/test_platform_settings.py tests/security/test_auth.py`, `pytest`, `ruff check .`
- **Validation:** `uv sync --dev` 후 ORM 모델 9개 테이블 import와 `alembic history`를 확인했다. 기존 런타임 경로는 JSON 저장소를 계속 사용하며 전체 백엔드 테스트 552개가 통과했다.
