# v3.2.0 Multicompany Admin Scope

## Background
The v3.1.0 admin console exposed operations data through a single
`app_metadata.role = admin` gate. PAZULE already had `organizations`,
`organization_members`, and `sites` in the DB model, but admin reads were not
scoped by company membership.

## Goal
Support B2B multicompany operations where platform masters can see every
company and company admins can see only their own organization's activity.

## Proposed Change
- Treat Supabase `app_metadata.role in {"admin", "platform_master"}` as
  platform-master access.
- Resolve organization-scoped admin access through `organization_members`.
- Add an admin organization listing endpoint for company filters.
- Scope admin summary, mission logs, coupon lists, user lists, and mission
  detail reads by organization visibility.
- Add a shared frontend company selector for admin pages.

## Impact
- Existing `admin` accounts keep full master visibility for compatibility.
- Non-master organization members can access admin data only for their company.
- JSON storage fallback remains master-only for broad admin reads because JSON
  records do not carry enough organization metadata for safe company scoping.
