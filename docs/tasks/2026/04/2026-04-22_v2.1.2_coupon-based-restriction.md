# 📋 작업 명세서: 쿠폰 기반 미션 접근 제한 강화

## 1. 배경 및 목적
- **배경:** 사용자가 미션 성공 후 쿠폰을 발급받았음에도 불구하고, 특정 상황에서 홈 화면의 미션 카드가 여전히 활성화되어 중복 시도를 유도하는 현상 방지 필요.
- **목적:** 당일 성공 여부뿐만 아니라 '쿠폰 보유 여부'를 완료 판정의 절대 기준으로 추가하여 미션 중복 시도를 원천 차단.

## 2. 구현 세부 사항
- [x] **백엔드 (`coupon_service.py`):** 특정 사용자(`user_id`)가 특정 정답(`answer`)에 대한 쿠폰을 이미 가지고 있는지 확인하는 `has_coupon_for_answer` 메서드 추가 완료.
- [x] **백엔드 (`routes.py`):** 
    - `/get-today-hint` API 로직 수정.
    - 이제 '당일 세션 성공' 또는 '이미 쿠폰 보유' 중 하나라도 해당되면 `completed: true`를 반환하도록 완료 판정 로직 강화 완료.

## 3. 검증 결과 및 품질
- **확인:** 과거에 성공하여 쿠폰을 보유 중인 랜드마크 미션의 경우, 홈 화면 로딩 시 즉시 '완료' 상태로 표시되어 진입이 차단됨을 확인.
- **확인:** 보상 획득(쿠폰 발급)이 미션 완수의 최종 정의임을 시스템적으로 명확히 함.

## 4. 핸드오버 참고 사항
- **정책:** 본 로직은 사용자 ID(`guest`)와 미션 정답 키워드를 기준으로 동작하므로, 쿠폰을 삭제하지 않는 한 동일 정답 미션에 대한 재참여는 불가능함.
