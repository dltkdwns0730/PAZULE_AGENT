# v3.3.2 README Hub Rewrite

## Background
The root README had grown into a long mixed document containing product
overview, architecture detail, environment setup, B2B operations, and resources.
Several sections were too detailed for first-time readers.

## Goal
Make the root README useful for interviewers, reviewers, general users, and
business readers by showing only essential project value, architecture, run
commands, verification commands, and document links up front.

## Proposed Change
- Replace the root README with a concise hub-style version.
- Keep first-read sections focused on what PAZULE is, why it matters, what it
  does, and how it is verified.
- Move detailed technical and operational paths to docs links instead of
  embedding long instructions.

## Validation Plan
- Check README section structure and total length.
- Confirm referenced local documents exist.
- Search for mojibake/corrupted text in the new README.

## Actual Results
- README reduced to a concise hub.
- All linked project documents were checked after replacing the missing
  `.env.example` reference with the existing config documentation path.
- No mojibake markers were found in README.

## Rollback
Restore the previous long README if a detailed single-file overview is needed
again.
