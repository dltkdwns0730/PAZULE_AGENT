# 📋 작업 명세서: 쿠폰 사용자 ID 연동 및 지갑 동기화 최종 수정

## 1. 배경 및 목적
- **배경:** 쿠폰 발급 성공 후에도 지갑 페이지에서 쿠폰이 보이지 않는 문제 발생. 조사 결과 발급된 쿠폰 데이터에 `user_id`가 누락되어 필터링 과정에서 제외됨을 확인.
- **목적:** 쿠폰 데이터 구조에 `user_id`를 추가하고 발급 시 이를 저장하도록 수정하여 지갑 페이지와의 완벽한 데이터 동기화 구현.

## 2. 구현 세부 사항
- [x] `app/services/coupon_service.py`: 
    - `issue_coupon` 메서드가 `user_id`를 인자로 받아 쿠폰 객체에 저장하도록 수정.
- [x] `app/api/routes.py`:
    - 쿠폰 발급 API(`/api/coupon/issue`) 호출 시 해당 미션 세션에 등록된 실제 `user_id`를 `issue_coupon` 서비스에 전달하도록 로직 수정.
- [x] `front/src/hooks/useMission.js`: 
    - `issueCoupon` 함수 내에서 발급 성공 시 전역 상태(`Store`) 업데이트를 명시적으로 보장하도록 재확인.

## 3. 검증 결과 및 품질
- **확인:** 신규 발급된 쿠폰의 `coupons.json` 데이터 내에 `user_id: "guest"` (또는 실제 ID) 필드가 포함됨을 확인.
- **확인:** 지갑 페이지(`/wallet`) 진입 시 해당 `user_id`로 필터링된 쿠폰 목록이 지연 없이 정상적으로 표시됨.

## 4. 핸드오버 참고 사항
- **주의:** 기존에 `user_id` 없이 발급된 구버전 쿠폰들은 지갑에서 보이지 않을 수 있으며, 신규 발급 건부터 정상 동작함.
